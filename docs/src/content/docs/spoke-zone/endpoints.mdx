## Retry Policy

- Applies to all auth and endpoint calls on this page
- Retry on transport errors, `429`, and `5xx`
- Do not retry `4xx` responses other than `429`
- Retry delays are fixed: `15s`, then `30s`, then `60s`
- If all retries fail, surface a typed SDK error for retry exhaustion with last-attempt diagnostics

## Authentication

- `DeviceAuth.login` and `UserAuth.login` use no auth header.
- All other endpoints on this page require `x-access-token: <token>`.

## DeviceAuth.login

`POST /loginDevice`

| Item | Contract |
| --- | --- |
| Success status | `200` or `201` |
| Error statuses to map | `400`, `401`, `403`, `429`, `5xx` |

### Request Body

- JSON object with `token` (string), `cpu_id` (string), `uuid` (string)

### Success Body

- `200`: `{ status, message }`
- `201`: `{ token }`

## UserAuth.login

`POST /login`

| Item | Contract |
| --- | --- |
| Success status | `200` |
| Error statuses to map | `401` (missing username/password or invalid credentials), `403` (account locked), `429`, `5xx` |

### Request Body

- JSON object with `username` (string) and `password` (string)

### Success Body

- `token` (string)
- `expires` (number)
- `user` object:
  - `username`
  - `user_id`
  - `user_role`
  - `customer_id`
  - `customer_role`
  - `oem_id`
  - `reset_required`
  - `customer_prefix`
  - `is_mobile_user`
  - `prefix`
  - `mqtt_permission`

## DataFiles.upload

`POST /api/v2/data-files/{id}/file`

| Item | Contract |
| --- | --- |
| Success status | `200` |
| Error statuses to map | `400`, `401`, `403`, `429`, `5xx` |

### Path Parameters

- `id` (number)

### Request Body

- `multipart/form-data` with `files` (binary) populated from raw bytes

### Upload Semantics

- Input is fully buffered bytes (`Uint8List`), encoded as multipart field `files`
- SDK does not provide chunked upload in this scope
- SDK retries per global retry policy; caller controls higher-level concurrency

## OtaFiles.list

`GET /api/v2/ota-files`

| Item | Contract |
| --- | --- |
| Success status | `200` |
| Error statuses to map | `400`, `401`, `403`, `429`, `5xx` |

### Query Parameters

- Supports documented pagination/search/sort/filter params (`searchTerm`, `searchFields`, `sort`, `sortOrder`, `limit`, `offset`)

### Success Body

- list of items with:
  - `id` (number)
  - `modelId` (number)
  - `moduleId` (number)
  - `module` (string)
  - `version` (string)
  - `fileLocation` (string)
  - `isActive` (boolean)
  - `createdDate` (string)
  - `releaseNotes` (string)

### List Defaults

- When omitted by caller: `limit=50`, `offset=0`
- Caller-provided query values override defaults

## Devices.get

`GET /api/v2/devices/{id}`

| Item | Contract |
| --- | --- |
| Success status | `200` |
| Error statuses to map | `400`, `401`, `403`, `404`, `429`, `5xx` |

### Path Parameters

- `id` (number)

### Success Body

- `id` (number)
- `identifier` (string)
- `serialNumber` (string)
- `modelId` (number)
- `modelName` (string, mapped from API field `name`)
- `lastOnline` (`DateTime?`, null when missing/invalid)
- `lastLocation` (`{ latitude: double, longitude: double }?`, null when either coordinate is missing)
- `softwareVersions` (`Map<String, String>`, empty map when missing/null)
